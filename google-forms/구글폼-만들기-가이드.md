# 구글 폼 만들기 가이드 (3탭 · 4명 제출용)

응답은 **Google 스프레드시트**에 자동 저장됩니다.  
추가로 **Git TXT** · **텔레그램 알림** 은 `FormResponseToGit.gs` 로 설정할 수 있습니다.

---

## 방법 A — 자동 생성 (추천, 5분)

### 1. 스크립트 열기

1. 브라우저에서 [https://script.google.com](https://script.google.com) 접속
2. **새 프로젝트**
3. `CreateMarketingForms3Tabs.gs` 파일 **전체 복사** → 붙여넣기
4. 기본 파일 `Code.gs` 내용은 **전부 지우고** 붙여넣기

### 2. 배우명 수정

맨 위 3줄만 바꿉니다.

```javascript
var ACTOR_NAME = '최지수';  // 실제 배우명으로 수정
var DEADLINE = '2026년 6월 30일';
var CONTACT = '홍길동 / echo@company.com';
```

### 3. 저장 후 실행

1. **Ctrl + S** 저장 (프로젝트 이름: `마케팅플랜 폼`)
2. 상단 함수 선택: **`createAllThreeForms`**
3. **실행(▶)** 클릭
4. **처음 1회:** 권한 검토 → Google 계정 선택 → **허용**
5. **실행 → 로그** (또는 보기 → 로그) 열기

로그 예시:

```
=== 구글 폼 3개 생성 완료 ===
① PR·마케팅·세일즈·콘텐츠
https://docs.google.com/forms/d/xxxxx/viewform
② 팬미팅
https://...
③ 스타일·로드맵·작품
https://...
```

### 4. Google Drive에서 확인

**내 드라이브**에 폼 3개가 생깁니다.  
각 폼 → **응답** 탭 → **스프레드시트에 링크** (없으면 클릭해서 생성)

### 5. 4명에게 보낼 링크

| 담당 | 보낼 폼 |
|------|---------|
| PR / 마케팅 / 세일즈 / 콘텐츠 | ① 번 URL |
| 팬미팅 | ② 번 URL |
| 매니지먼트 / 스타일 / 작품 | ③ 번 URL |

한 사람이 여러 영역이면 **해당 폼만** 여러 번 제출.

## 방법 B — 직접 만들기 (클릭만)

1. [https://forms.google.com](https://forms.google.com) → **빈 양식**
2. 제목: `[배우명] ① PR·마케팅·세일즈·콘텐츠`
3. 설명에 회신 기한·에코 문의 적기
4. 질문 추가:
   - **단답형:** 소속사, 작성자, 키워드…
   - **장문형:** 일정, 보도 계획…
   - **선택형:** 팬미팅 계획…
5. **섹션 나누기**로 PR / 마케팅 / 세일즈 / 콘텐츠 구분
6. **응답** 탭 → 스프레드시트 연결
7. ② 팬미팅, ③ 로드맵·작품 폼도 같은 방식으로 **새 폼 2개** 추가 생성

질문 목록은 `marketing-intake/index.html` 또는 아래 요약 참고.

## 응답 모으기

### 스프레드시트 3개 → 1개로 합치기 (선택)

1. Google Drive에서 **새 스프레드시트** `배우명_마케팅플랜_응답`
2. 시트 탭 이름: `PR마케팅`, `팬미팅`, `로드맵작품`
3. 각 폼의 응답 시트에서 **복사 → 붙여넣기**
4. 또는 메뉴 **데이터 → 시트 가져오기**

### 알림 받기

각 폼 → **응답** → ⋮ → **새 응답 시 이메일 알림 받기**

## 제출 시 TXT 저장 · 텔레그램 (선택)

`FormResponseToGit.gs` 한 파일로 **둘 다 또는 하나만** 켤 수 있습니다.

파일 맨 위 `NOTIFY` 설정:

```javascript
var NOTIFY = {
  saveToGit: true,      // GitHub responses/ 에 .txt
  sendTelegram: true,   // 내 텔레그램으로 전체 내용
};
```

### Git에 자동 저장 (HTML이랑 같은 TXT 형식)

구글 폼으로 받으면서 **제출할 때마다 Git `responses/` 에 TXT** 로 쌓을 수 있습니다.

#### 설정 (1회)

1. `FormResponseToGit.gs` 를 **같은 Apps Script 프로젝트**에 붙여넣기
2. `GIT_CONFIG` 수정:
   ```javascript
   var GIT_CONFIG = {
     token: 'ghp_xxxx',  // GitHub 토큰 (repo 쓰기 권한)
     repo: '아이디/저장소이름',
     branch: 'main',
   };
   ```
3. 함수 **`setGitConfig`** 실행
4. (텔레그램 쓸 때) `TELEGRAM_CONFIG` 수정 후 **`setTelegramConfig`** 실행
5. 함수 **`installFormTriggers`** 실행
6. GitHub에서 `responses/` 폴더 확인 (첫 제출 후 파일 생성)

### 텔레그램으로 전체 내용 받기

1. [@BotFather](https://t.me/BotFather) 에서 봇 생성 → **bot token** 복사
2. 봇과 대화 한 번 시작 (채팅방 열기)
3. [@userinfobot](https://t.me/userinfobot) 등으로 본인 **chat_id** 확인
4. 스크립트에 입력:
   ```javascript
   var TELEGRAM_CONFIG = {
     botToken: '123456:ABC...',
     chatId: '123456789',
   };
   ```
5. **`setTelegramConfig`** 실행 → **`testTelegram`** 실행 (테스트 1통)
6. **`installFormTriggers`** (아직 안 했다면)

제출 시 동작:

- **`.txt` 파일**이 텔레그램으로 첨부됨 (내용 전체)
- 짧은 안내 문구(배우명·탭·작성자)가 캡션으로 붙음
- Git만 끄려면 `NOTIFY.saveToGit = false`
- 텔레그램만 쓰려면 `NOTIFY.sendTelegram = true`, Git 설정 생략 가능

### 기존 시트 응답도 Git에 올리기

함수 **`exportAllFormsToGit`** 실행 → 지금까지 제출분 한꺼번에 업로드

### 저장 경로 예시

```
responses/
  └── 최지수/
        └── 2026-05-20T11-30-00_PR마케팅세일즈콘텐츠_홍길동.txt
```

시트에도 그대로 쌓이고, Git에는 **읽기 좋은 TXT** 로도 쌓입니다.
